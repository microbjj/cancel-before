# GitFlow для CancelBefore

Две ветки: **main** и **develop**.

## Ветки

| Ветка       | Назначение |
|------------|------------|
| **develop** | Ведётся вся разработка. Пушите сюда коммиты. |
| **main**    | Продакшен. Сюда попадает только проверенный код через PR. |

## Как работать

1. **Разработка** — работаете в ветке **develop**, пушите в `origin develop`.
2. **Когда готово к релизу** — в GitHub создаёте **Pull Request: develop → main**.
3. **CI должен быть зелёным** (проверки запускаются на PR автоматически).
4. После одобрения и мержа PR код из develop попадает в main.

Схема:

```
develop  ----*---*---*---*---*---*---*---*  (разработка, push сюда)
                    \
                     \  PR (когда готово, CI прошёл)
                      \
main     --------------*------------------  (релиз после мержа PR)
```

## CI

При пуше и при создании PR в ветки **main** и **develop** запускается проверка: lint, typecheck, тесты, сборка. В main мержим только когда на PR всё зелёное.
